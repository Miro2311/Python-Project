Properties (Getter/Setter)

üéØ Was sind Properties?

Properties sind eine elegante Art, den Zugriff auf Attribute zu kontrollieren. Sie lassen normale Methodenaufrufe wie Attributzugriffe aussehen. Ein Getter wird beim Lesen eines Attributs aufgerufen, ein Setter beim Schreiben.

ü§î Warum Properties verwenden?

  • Datenvalidierung beim Setzen
  • Berechnete Werte (read-only)
  • Logging/Debugging
  • Backwards compatibility

‚öñÔ∏è Unterschied Attribut vs Property:

  • Attribut: Direkter Zugriff auf gespeicherten Wert
  • Property: Methodenaufruf, der wie Attributzugriff aussieht
class Circle:
    def __init__(self, radius):
        self._radius = radius  # "Private" Attribut (Konvention mit _)
    
    @property
    def radius(self):
        """Getter - wird bei obj.radius aufgerufen"""
        print("Radius wird gelesen")
        return self._radius
    
    @radius.setter
    def radius(self, value):
        """Setter - wird bei obj.radius = value aufgerufen"""
        print(f"Radius wird gesetzt auf {value}")
        if value < 0:
            raise ValueError("Radius muss positiv sein")
        self._radius = value
    
    @property
    def area(self):
        """Read-only Property - kein Setter definiert"""
        return 3.14159 * self._radius ** 2
    
    @property
    def diameter(self):
        """Berechnet aus radius"""
        return 2 * self._radius

# Verwendung
circle = Circle(5)
print(circle.radius)    # Getter aufgerufen: "Radius wird gelesen" -> 5
circle.radius = 10      # Setter aufgerufen: "Radius wird gesetzt auf 10"
print(circle.area)      # 314.159 (automatisch berechnet)
print(circle.diameter)  # 20 (automatisch berechnet)

# circle.area = 100     # AttributeError! (Kein Setter definiert)

class Car:

class Animal:  # Superclass/Parent class
    def __init__(self, name, species):
        self.name = name
        self.species = species
    
    def speak(self):
        return "Some generic animal sound"
    
    def info(self):
        return f"{self.name} is a {self.species}"

class Dog(Animal):  # Subclass/Child class - Dog IS an Animal
    def __init__(self, name, breed):
        super().__init__(name, "Canis lupus")  # Ruft Parent __init__ auf
        self.breed = breed
    
    def speak(self):  # Override parent method
        return f"{self.name} sagt Wuff!"
    
    def fetch(self):  # Neue Methode nur in Dog
        return f"{self.name} brings the ball back"

class Cat(Animal):  # Cat IS an Animal
    def speak(self):  # Override parent method
        return f"{self.name} sagt Miau!"
    
    def climb(self):  # Neue Methode nur in Cat
        return f"{self.name} climbs the tree"

# Verwendung
